## DESARROLLO WEB CON FLASK

**Objetivo**

Desarrollar un sistema web utilizando Python, Flask y SQLAlchemy, implementando
autenticación de usuarios, relaciones entre tablas, reportes con gráficas y trabajo colaborativo
mediante GitHub.

**Requerimientos del Proyecto**

**1. Trabajo en Equipo**

El proyecto deberá realizarse en equipos de:

- Mínimo: 2 integrantes
- Máximo: 3 integrantes
**2. Requerimientos Técnicos**

El proyecto deberá utilizar:

- Python
- Flask
- SQLAlchemy
- GitHub
**3. Base de Datos**

El sistema deberá contener:

- Mínimo 4 tablas relacionadas
- Uso correcto de:
    o ForeignKey
    o relationship
    o back_populates


Ejemplo:

- Usuario
- Producto
- Categoría
- Venta
- DetalleVenta
**4. Autenticación**

El sistema deberá incluir autenticación funcional.

**Requisitos** :

- Inicio de sesión (Login)
- Cierre de sesión (Logout)
- Protección de vistas o módulos
- Manejo de usuarios mediante Flask-AppBuilder

Se evaluará:

- Funcionamiento correcto del login
- Restricción de acceso a usuarios no autenticados
**5. CRUD Completo**

El sistema deberá permitir:

- Crear registros
- Listar registros
- Editar registros
- Eliminar registros

Cruds por cada tabla que se tenga en el proyecto


**6. Reportes**

El sistema deberá incluir:

- Un módulo de reportes
- Una tabla principal para generar estadísticas
- Consultas con:
    o conteos
    o sumatorias
    o agrupaciones
**7. Gráficas**

El proyecto deberá contener mínimo:

- 1 gráfica dinámica

Ejemplos:

- Barras
- Pastel
- Líneas

Las gráficas deberán mostrar información almacenada en la base de datos.

**8. GitHub y Trabajo Colaborativo**

Requisitos Obligatorios

Todo el proyecto deberá subirse a GitHub.

Cada integrante debe:

- Trabajar utilizando ramas (branches)
- Realizar commits propios
- Subir avances constantemente


- Participar activamente en el repositorio

Se Revisará

- Commits
- Uso correcto de branches
- Participación individual dentro del proyecto
- Organización del repositorio
**9. Presentación Final**

Lunes 18 de mayo

**10. Registro de Presentaciones**

Todos los grupos deberán registrarse en el enlace adjunto para definir el orden de exposición.

**11. Exposición**

Durante la presentación deberán explicar:

- Nombre del proyecto
- Problemática que resuelve
- Funcionamiento del Sistema autenticación los CRUDS y reporte más grafica
